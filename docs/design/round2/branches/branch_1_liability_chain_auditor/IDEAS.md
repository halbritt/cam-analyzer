# Liability Chain Auditor Ideas

- **Custody ledger per returned value.** Every `ProfileResult` includes the last accountable decision owner, not just data provenance: parser, unit normalizer, interpolation operator, derivative policy, confidence downgrader, refusal gate, or analysis threshold.

- **Decision handoffs require explicit acceptance.** A source adapter cannot silently hand a weak cam-card approximation to a canonical operator; the operator records `accepted_from`, `accepted_as`, and the exact assumptions it now owns.

- **Refusal names the missing owner.** When `CamProfile` cannot answer a query at the requested fitness, the refusal identifies the absent accountable party, such as `seat-ramp measurement owner`, `seam-continuity prover`, or `high-order derivative policy owner`.

- **Threshold policies own cliff verdicts.** PTV contact, coil-bind, spring-float, and retainer-guide results name the analysis policy that converted a continuous value into a safe/unsafe/refused verdict, separate from the profile owner of the curve region.

- **Magnitude extraction becomes a custody transfer.** Any `.magnitude` or bare-float export stamps the caller as the new accountable owner until provenance and units are reattached through a formal constructor.

- **Reports print the broken chain.** If accountability evaporates between `CamCardApproxProfile` and an analysis module, the report shows the exact vanished handoff and the last named owner instead of smoothing it into generic low confidence.
